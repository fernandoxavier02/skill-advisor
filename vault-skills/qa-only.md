# QA Only

## Conceitos
- [[structured-testing]] — organized test methodologies
- [[report-generation]] — detailed QA findings documentation
- [[health-scoring]] — quantitative quality assessment
- [[evidence-collection]] — screenshots and reproducible steps
- [[issue-categorization]] — severity and impact ranking

## Workflow
1. Systematically test web application
2. Document all findings with evidence
3. Categorize issues by severity
4. Generate structured report
5. Calculate overall health score
6. Provide improvement recommendations

## Conecta com
- [[qa]] expands: with automated fixing
- [[investigate]] debugs: reported issues
- [[health]] correlates: with discovered issues

## Quando usar
- Generate QA reports without fixing
- Stakeholder communication on quality
- Handoff to engineering team
- Documentation purposes
- When you want findings only (no implementation)
- Risk assessment before release
