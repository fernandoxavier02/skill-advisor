# GStack OpenClaw CEO Review

## Conceitos
- [[planning]] — applies founder-mode thinking to evaluate plans beyond technical correctness
- [[architecture]] — challenges architectural decisions from a product-market fit perspective
- [[quality]] — elevates quality bar by asking "what would make this 10x better"

## Workflow
1. Receive and analyze the implementation plan or design document
2. Rethink the problem from first principles, challenging all premises and assumptions
3. Identify the "10-star product" version and expand scope where it creates value
4. Deliver critical review with actionable recommendations

## Conecta com
- [[write-plan]] recebe: revised plan with CEO-level insights
- [[brainstorm]] recebe: expanded scope ideas and 10-star vision
- [[code-architect]] recebe: challenged architectural premises with alternative approaches

## Quando usar
- Before committing to a major feature or architectural direction
- When a plan feels "good enough" but you suspect it could be significantly better
- When you need a fresh, high-level perspective that questions fundamental assumptions
