# QA

## Conceitos
- [[systematic-testing]] — structured QA methodology
- [[bug-discovery]] — finding issues in production
- [[regression-testing]] — ensuring existing features still work
- [[bug-fixing]] — resolving discovered issues
- [[test-iteration]] — continuous improvement cycle

## Workflow
1. Systematically test web application
2. Document bugs found with reproducible steps
3. Fix bugs in source code
4. Commit changes
5. Re-test to verify fixes
6. Iterate until application is stable

## Conecta com
- [[qa-only]] provides: testing without fixes
- [[investigate]] debugs: complex issues
- [[review]] validates: code quality of fixes

## Quando usar
- Before shipping to production
- After major feature release
- During QA phases
- Regression testing after updates
- When application feels unstable
- Full test cycle on critical features
